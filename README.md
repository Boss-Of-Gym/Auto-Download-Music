# AutoDownload

Программа для автоматического скачивания MP3-треков по списку. Поддерживает два источника: готовый JSON-файл со списком треков или плейлист из Яндекс.Музыки.

## Что умеет программа

- Берёт список треков в формате `Исполнитель — Название`
- Автоматически ищет каждый трек на музыкальных сайтах (zaycev.net, hitmoz.org)
- Скачивает найденные треки в MP3-формате в выбранную папку
- Запоминает прогресс — при повторном запуске пропускает уже скачанные треки
- Поддерживает параллельное скачивание нескольких треков одновременно
- Может забрать список треков прямо из плейлиста Яндекс.Музыки
- Показывает оставшееся время загрузки прямо в статусной строке
- Уведомляет звуком и выдвигает окно на передний план по завершении
- Клик по строке трека — копирует название в буфер обмена
- Запоминает размер и положение окна между запусками

## Системные требования

- Windows 10 или Windows 11
- ~500 МБ свободного места (135 МБ программа + ~180 МБ браузер Chromium при первом запуске)
- Стабильное интернет-подключение
- Python **не требуется** — всё включено в дистрибутив

## Установка

1. Скачайте папку `AutoDownload\` и скопируйте её в любое удобное место, например:
   ```
   C:\Users\ВашеИмя\Desktop\AutoDownload\
   ```
   или
   ```
   D:\Programs\AutoDownload\
   ```

2. Никаких дополнительных установок не требуется. Все зависимости уже внутри папки.

> **Важно:** не переносите только файл `AutoDownload.exe` отдельно — программа работает только вместе со всей папкой `AutoDownload\`.

## Первый запуск

### Шаг 1 — Запуск

Дважды щёлкните на `AutoDownload.exe`. Откроется окно программы.

### Шаг 2 — Загрузка браузера (только один раз)

При первом запуске программа автоматически скачает встроенный браузер Chromium (~180 МБ). Это происходит один раз и занимает 1–3 минуты в зависимости от скорости интернета.

В этот момент вы увидите в строке статуса сообщение об установке браузера. **Не закрывайте программу** — дождитесь завершения.

После этого браузер сохранится в папке `AutoDownload\browsers\` и при следующих запусках скачивание не повторяется.

### Шаг 3 — Готово

После загрузки браузера программа полностью готова к работе.

## Интерфейс программы

Окно разделено на две части:

```
┌─────────────────────┬──────────────────────────────────┐
│  ↓ AutoDownload     │  Прогресс              3 / 10    │
│  Поиск и загрузка   │  Обработано 3 из 10 · ~2 мин     │
│                     │                                  │
│ ▌ ИСТОЧНИК          │  ▌ Кино — Группа крови     ✓    │
│  [JSON] [Яндекс]    │  ▌ Nirvana — Come As You ⣾     │
│                     │  ▌ Linkin Park — In The End  ·   │
│ ▌ ПАПКА ЗАГРУЗОК    │  ...                             │
│ ▌ ПАРАЛЛЕЛЬНЫХ П.   │                                  │
│ ▌ МАКСИМУМ ТРЕКОВ   │  ✓ 1    ◌ 1    ✗ 0              │
│                     │  ══════════════════              │
│  [▶ Запустить]  [📂]│                                  │
│  Enter · Esc        │                                  │
└─────────────────────┴──────────────────────────────────┘
```

### Левая панель — настройки

| Элемент | Описание |
|---|---|
| **ИСТОЧНИК** | Переключатель между режимами: `JSON файл` или `Яндекс.Музыка`. Под переключателем — краткое описание выбранного режима |
| **ПАПКА ЗАГРУЗОК** | Куда сохранять MP3-файлы. Кнопка `…` открывает проводник |
| **ПАРАЛЛЕЛЬНЫХ ПОТОКОВ** | Сколько треков искать одновременно (1–10). Подробнее — в разделе [настроек](#дополнительные-настройки) |
| **Запускать браузер в фоне** | Если включено — браузер работает в фоне невидимо |
| **МАКСИМУМ ТРЕКОВ** | Оставьте пустым чтобы скачать все, или введите число |
| **▶ Запустить** | Кнопка запуска. Во время работы становится **■ Стоп** |

### Правая панель — прогресс

Здесь отображается каждый трек со статусом. Строка состоит из трёх элементов:

```
  ▌  ⣾  Кино — Группа крови
  │  │  └─ название трека
  │  └──── иконка статуса
  └─────── цветная полоска (статус)
```

| Полоска | Иконка | Статус |
|---|---|---|
| серая | `·` | Ожидает обработки |
| жёлтая | `⣾` (анимация) | Идёт поиск |
| зелёная | `✓` | Успешно скачан |
| красная | `✗` | Не найден ни на одном сайте |

**Клик по строке** копирует название трека в буфер обмена — строка мигнёт в подтверждение.

Внизу правой панели — счётчики (в виде цветных меток) и полоса прогресса.

### Статусная строка

Во время загрузки показывает:
```
Обработано 5 из 20  ·  ~3 мин
```

По завершении:
```
Готово  •  Скачано: 18  •  Не найдено: 2
```

### Уведомление о завершении

Когда загрузка заканчивается (даже если окно свёрнуто или на заднем плане):
- Программа воспроизводит системный звуковой сигнал
- Окно автоматически появляется поверх других окон на 3 секунды

## Режим: JSON-файл

Используйте этот режим если у вас уже есть список треков.

### Формат файла

Создайте текстовый файл и сохраните его с расширением `.json`. Содержимое:

```json
[
  {"index": 1, "track": "Кино — Группа крови", "status": "pending"},
  {"index": 2, "track": "Nirvana — Smells Like Teen Spirit", "status": "pending"},
  {"index": 3, "track": "Linkin Park — In The End", "status": "pending"}
]
```

**Правила написания треков:**
- Формат: `Исполнитель — Название` (тире с пробелами)
- Регистр не важен
- Можно писать только название без исполнителя — программа всё равно попробует найти

### Как указать файл в программе

**Вариант А — путь к файлу:**
Нажмите `…` и выберите ваш JSON-файл через проводник, или вручную введите полный путь:
```
C:\Users\ВашеИмя\Documents\my_playlist.json
```

**Вариант Б — имя плейлиста:**
Если вы ранее запускали режим Яндекс.Музыки, программа уже создала JSON-файл рядом с собой. Введите просто имя плейлиста (например `Мне нравится`) — программа найдёт файл сама.

## Режим: Яндекс.Музыка

Программа откроет браузер, войдёт в ваш аккаунт Яндекс.Музыки, соберёт треки из указанного плейлиста и запустит скачивание.

### Необходимые данные

Поля, отмеченные `*`, обязательны. Если оставить их пустыми и нажать **▶ Запустить**, под незаполненным полем появится подсказка с описанием ошибки, а граница поля станет красной.

| Поле | Обязательное | Что вводить | Пример |
|---|---|---|---|
| **Телефон \*** | Да | Номер телефона аккаунта Яндекс.ID | `+79991234567` |
| **Логин \*** | Да | URL-ник — виден в адресной строке на странице профиля | `ivanov_ivan` |
| **Имя в шапке \*** | Да | Отображаемое имя аккаунта на сайте | `Иван Иванов` |
| **Плейлист \*** | Да | Точное название плейлиста как в Яндекс.Музыке | `Мне нравится` |

> **Где найти URL-ник:** зайдите на [music.yandex.ru](https://music.yandex.ru), нажмите на своё имя. В адресной строке будет: `music.yandex.ru/users/`**ivanov_ivan**`/playlists` — это и есть логин. В поле **Логин** введите: `ivanov_ivan`.

### Прогресс сбора плейлиста

Во время подключения и сбора треков из Яндекс.Музыки программа отображает прогресс в правой панели:

- Полоса прогресса **анимируется** (движется туда-обратно) — идёт подключение или начальный сбор
- Как только количество треков становится известно, прогресс переключается в режим `N / M` с реальным процентом заполнения
- Статусная строка показывает: `Сбор треков: 47 / 120`
- Как только все треки собраны и скачивание началось, полоса отражает реальный прогресс загрузки

### Авторизация и OTP-код

При запуске программа откроет браузер и попытается войти в аккаунт. Яндекс.ID отправит **push-уведомление** на ваш телефон с 6-значным кодом подтверждения.

Когда код будет отправлен, программа автоматически покажет диалоговое окно:

```
┌─────────────────────────────────────┐
│  Яндекс ID — подтверждение          │
│                                     │
│  Push-уведомление отправлено на     │
│  номер телефона.                    │
│                                     │
│  Введите 6-значный код:             │
│  [ _________ ]                      │
│                                     │
│        [ OK ]  [ Отмена ]           │
└─────────────────────────────────────┘
```

Введите код из уведомления и нажмите **OK**. Программа продолжит работу автоматически.

### Что происходит после авторизации

1. Программа прокручивает плейлист и собирает все треки
2. Сохраняет список в файл `ИмяПлейлиста_tracks.json` рядом с `AutoDownload.exe`
3. Начинает поиск и скачивание

## Процесс скачивания

### Как работает поиск

Для каждого трека программа:

1. Открывает сайт-источник (zaycev.net или hitmoz.org)
2. Вводит запрос `Исполнитель — Название`
3. Сравнивает результаты поиска с нужным треком
4. Если совпадение найдено — скачивает MP3
5. Если на первом сайте не нашлось — пробует следующий
6. Если ни один сайт не дал результата — помечает трек как `не найден`

### Куда сохраняются файлы

Треки сохраняются в папку, указанную в поле **ПАПКА ЗАГРУЗОК**. По умолчанию это:
```
C:\Users\ВашеИмя\Music\
```

Имя файла соответствует названию трека как оно найдено на сайте.

### Не найденные треки

После завершения, если часть треков не удалось найти, внизу панели прогресса появится кнопка **Скопировать ненайденные в буфер** — она мигнёт чтобы привлечь внимание. Нажмите её чтобы скопировать список ненайденных треков в буфер обмена и сохранить вручную или поискать на других ресурсах.

Также можно кликнуть на любую строку с `✗` в списке — название трека скопируется в буфер.

### Файл прогресса

Рядом с `AutoDownload.exe` создаётся JSON-файл вида `ИмяПлейлиста_tracks.json`. Он хранит статус каждого трека:

- `pending` — ещё не обработан
- `downloaded` — успешно скачан
- `not_found` — не найден ни на одном сайте

Этот файл позволяет **возобновлять** прерванные загрузки.

## Прерывание и возобновление

Если вы закрыли программу во время скачивания — ничего страшного.

При следующем запуске:
1. Укажите тот же источник (JSON-файл или тот же плейлист Яндекс.Музыки)
2. Нажмите **▶ Запустить**
3. Программа прочитает файл прогресса и пропустит уже скачанные треки
4. Обработка продолжится только с треков в статусе `pending`

> **Для Яндекс.Музыки:** при повторном запуске введите имя плейлиста в поле **JSON файл** (режим `JSON файл`) — программа найдёт уже созданный файл и не будет заново авторизовываться и парсить плейлист.

## Дополнительные настройки

### Параллельные потоки

Регулируется ползунком от 1 до 10.

| Значение | Когда использовать |
|---|---|
| **1–2** | Медленный интернет или нужна минимальная нагрузка |
| **3** | Рекомендуемое значение по умолчанию |
| **5–7** | Быстрый интернет, хотите ускорить загрузку |
| **8–10** | Максимальная скорость, но выше риск блокировки со стороны сайтов |

### Запускать браузер в фоне

Чекбокс **«Запускать браузер в фоне»**:

- **Снят (по умолчанию)** — вы видите браузер во время работы. Удобно для наблюдения за процессом.
- **Включён** — браузер работает полностью в фоне без видимого окна. **Не рекомендуется** при первом использовании режима Яндекс.Музыки — диалог ввода OTP-кода всё равно появится, но браузерное окно не будет отображаться.

### Максимум треков

Оставьте поле пустым — программа обработает все треки. Введите число (например `20`) — программа остановится после обработки первых 20 треков из списка.

### Размер и положение окна

Программа автоматически запоминает размер и положение окна при каждом закрытии и восстанавливает их при следующем запуске.

## Частые вопросы и проблемы

### Программа долго запускается в первый раз

Это нормально — идёт загрузка браузера Chromium (~180 МБ). Дождитесь завершения, не закрывайте программу.

### Трек помечен как «не найден» (✗)

Программа перебрала все доступные сайты и не нашла совпадения. Причины:
- Трек очень редкий или новый
- Написание исполнителя или названия не совпало с базой сайтов
- Сайт временно недоступен

Используйте кнопку **Скопировать ненайденные в буфер** чтобы получить полный список. Можно исправить написание трека в JSON-файле (поменяйте статус обратно на `pending`) и запустить снова.

### Браузер открывается и сразу закрывается при Яндекс.Музыке

Проверьте правильность номера телефона и логина. Логин — это именно URL-ник, а не email или номер.

### Много треков не находит, хотя они существуют

Попробуйте уменьшить количество параллельных потоков (до 2–3) — сайты могут блокировать слишком частые запросы.

### Ошибка при запуске: «Windows защитила ваш компьютер»

Нажмите **«Подробнее»** → **«Выполнить в любом случае»**. Это стандартное предупреждение Windows для программ без платной цифровой подписи.

### Куда сохраняется файл прогресса?

В папку рядом с `AutoDownload.exe`:
```
AutoDownload\
├── AutoDownload.exe
├── МойПлейлист_tracks.json   ← файл прогресса
└── ...
```

### Можно ли запустить несколько экземпляров одновременно?

Нет, не рекомендуется — оба экземпляра будут писать в один файл прогресса, что приведёт к конфликтам.
